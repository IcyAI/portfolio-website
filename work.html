MyFlix (REST API)

To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

The goal of the app is to complete the API side of the MyFlix project. The Myflix app will allow users to sign up, login in, view and sort movies, favorite and unfavorite movies, delete their account, and view information about selected movies, like director, genre, and summary. The API will create, read, update, and delete user and movie information from a database.

Technologies used: Vscode, HTML, CSS, JavaScript, Node.js, NPM, Express.js, SQL (practice), NoSQL, Heroku (API Host), MongoDB (Database Host), Mongoose, Postman

Meet App

The objective was to build a serverless, progressive web application (PWA) with React using a test-driven development (TDD) technique. The application uses the Google Calendar API to fetch upcoming events.

The goal of the app is to display different events going on in cities around the world. The user can sort the events by city and the number of events displayed. Each event has a description and the website offers two types of charts that display data about the events. The app can also be downloaded as a PWA from the browser.

Technologies used: Vscode, HTML, CSS, JavaScript, React, Recharts, Amazon Web Services Lambda, Google Calandar API, Google OAuth, Unit Tests, Inegration Tests, Acceptance Tests, End to End Tests, PWA Functionality, User Stories, Test Scenarios, GitHub, GitHub pages
